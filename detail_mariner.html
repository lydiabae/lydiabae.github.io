Mariner 7

Mariner 7 is an online performance management tool for small to large organisations. The tool enables employees to take ownership of their performance. Employees are able to prepare their professional goals or reviews and submit it to their manager to approve it. The tool encourages and supports managers and employees to involved in review process actively.

Skills used

  • User Experience
  • User Interface Design
  • Digital Branding
  • Art Direction

User types

Primary users are manager, employee and administrator. The manager’s role is to provide feedback, coaching and guidance to employee. Pain points: Number of employee, Multiple review & send task, Creating KA templates, Notifications, New role - My KAs, Employee KAs.

Employees are able to create goals, individual development plans and rate their scores. Pain points: New employee, Set up reminders, Understand rating scales.

Administrator is able to add or remove employees. Sometimes an administrator is entitle to set up the library templates for key roles. Pain points: Managing large teams, Organisation structure, Custom rating scales.

Key sections

I reviewed the key sections on exiting applications and prioritise all features. Blue sticky notes indicate the primary tasks or actions that need to be completed by users.

  • Dashboard: Existing tool doesn’t include a dashboard. A user can only browse other sections from Key Accountability section or a link from the notification email message. Dashboard offers a quick view on team member’s rating and review status.
  • Key Accountabilities: A manager and employee can set & agree goals and plan actions based on each goal. Key accountability section includes multiple competencies and goals. Manager can create, edit and remove them. The existing layout didn’t communicate effectively.
  • Performance Rating: The existing performance rating has a drop down selector to choose a rating scale. Sometimes rating scale can be more than 10.
  • Individual Development Plan: Individual Development Plan is a new section and it helps employees to create and plan their professional development.
  • Library: The library stores existing and new templates for Key Accountability and Performance rating templates. It’s managed by administrator and managers can create a new template or use existing ones for their reviews in editing mode. Existing library call to action isn't very prominent on detail sections.
  • Feedback: Feedback feature is designed for asking questions or requesting feedback on a team member or yourself. It’s a quick way of communicating between a manager and employee. The existing feedback process has more than 4 steps.
  • Others: The exisitng site has no maximum width and it's difficult to read the content in a wide screen. Some pages need typographic hierarchy to increase readability.
  • Gateway: Provides a quick status on an employee’s behaviour. It divided into three types: Significant gateway breach, Gateway issues identified and Adhering to policy.

User Journeys

During the workshop, we have quickly sketched the user journeys for each user type. Also we discovered that a manager’s flow can be change based on employee’s actions.

Wireframe - Dashboard

All sections have the primary call to actions on the header area. I have prioritised content types and divided them into four sections on dashboard. The tasks and team status are displayed on the left column and secondary and tertiary items are positioned at the right column. A manager can see ‘My Team’ status on the dashboard. A summary of team status helps a manager to identify urgent tasks.

Wireframe - Key sections

Key Accountability section contains large amount of content. Each competency can also contain multiple goal. Users can easily lose the focus while they are completing each task. Creating a better flow and typographic hierarchy is the most important task for this section. Secondary action panels can be collapsed. Using inline content editing helps users to click on less buttons while they are previewing or editing a specific content.

Review (Performance rating) section provides a better and easy way to score the employee’s performances and behaviours. Displaying sticky rating scale panel helps users to reference while they complete a long list of rating tasks. Also a user can easily change the rating by dragging or clicking on a scroller module.

Branding & Styleguide

Performance review process can be scary and stressful for employees and managers. I want to avoid the rigid and boring bits of review process and make it more fun and engaging for every user. I’ve chosen soft colour palette to make the interface more conversational and contemporary. Functional colours are used for validation, error, notification messages.

Typography

Slab serif is a certain kind of serif typeface that has thick, squared-off serifs/ Slab Serif fonts are more modern and bold and the lower case option is perfect for larger headings.

Iconography

I’ve decided to create a hand drawn icon set. This icons style delivers a fun and vibrant personality. My biggest challenge was developing hand drawn icons that work on a smaller size.

Page Layout

The tool is targeting for desktop users only. The layout needs to help the content highlighting what the most important pieces in that page. I started looking at the high level frame layout then added components.

Login/Request a new password

The login form and process should be simple and effortless as possible. I sketched out each steps and potential states(validations, suggestions) before designing them.

Onboarding

Designing an onboarding flow can be the most exciting task. The challenge is that the message need to be simple and clear as possible. The tool is limited to maximum 5 onboarding screens to explain the review process. A user can access this overview section from the dashboard.

Dashboard

Dashboard provides a quick overview on a user’s current tasks and the team status. The latest tasks are limited to 5 on default view and a scroll bar will be displayed on a longer listing.

Team status are labeled with colours and the filter can be applied on team listing.

Key Accountabilities

Competencies can include custom icons and multiple goals. Adding icon helps scalability. Mandatory field is clearly labeled on editing and non editing modes.

Goals can be edited individually (inline editing). Tooltip provides further explanations on specific terms.

Reviews

I created three simple step to explain the overall rating. Overall rating contains manager’s rating, employee’s rating, Adjustment rating and confirmed rating. On a default view, only manager and employee’s rating will be displayed on a slider. A user is able to add and remove a marker to adjust a rating.

Individual Development Plan

Donut graph timer shows remaining time until the completion. Employees are able to track their development plans and remaining times by looking at the timer and plan details.

My Team

This page will be defaulted to current user, showing own team members. User can navigate upper and lower level of team structure by clicking icons. eg. GO UP ONE LEVEL. + icon will show lower level team listings. - icon will hide it. Click on to team member’s name, it will bring up Team Listing Page (if this is a manager who has team members) or bring up profile page (if this is a employee)

Gateways

Colour labeled message is the key on Gateway section. A manager is able to add reason and comment based on an option.
